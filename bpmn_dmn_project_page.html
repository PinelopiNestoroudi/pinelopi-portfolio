Project Overview

The tool was built to support structured evaluation of process and decision models, not only by displaying diagrams, but also by turning them into analyzable objects with metrics, interpretive summaries, quality flags, and relative comparison outputs.

What the tool does

From uploaded models to structured evaluation

  • Accepts BPMN and DMN files and organizes them into analysis-ready models.
  • Supports BPMN-only, DMN-only, and composite BPMN+DMN model sets.
  • Computes structural and quality-oriented metrics for each model type.
  • Generates qualitative summaries and issue-oriented interpretation to make outputs more usable.
  • Provides comparison workflows where scores are normalized within the selected comparison set.
  • Supports BPMN–DMN mapping for combined analysis through a dedicated mapping editor.
  • Model analysis
  • Comparison workflows
  • Integration mapping
  • Interpretive summaries
Why it matters

More than a diagram viewer

The value of the project lies in combining diagram inspection with logic for evaluation and interpretation. Instead of stopping at visualization, the tool supports a fuller assessment workflow that helps users inspect quality, compare alternatives, and reason about BPMN–DMN integration in a more structured way.

Key technical ideas

Session-based workspaces, model creation flows, API-backed analysis panels, BPMN/DMN rendering, comparison scoring, and mapping persistence for composite models.

Key Capabilities

The project combines process-model analysis, decision-model analysis, and combined model evaluation in one interface, with a workflow designed for exploration rather than purely static reporting.

BPMN analysis

Process structure and quality indicators

Users can inspect BPMN diagrams, view structural metrics, identify modeling warnings, and receive qualitative annotations that help interpret complexity and understandability characteristics.

DMN analysis

Decision-model diagnostics

DMN models can be evaluated through metrics, quality flags, and interpretive summaries, including table structure, decision density, dependency depth, and maintainability-oriented observations.

Combined mode

BPMN–DMN integration mapping

Composite models connect BPMN Business Rule Tasks with DMN decisions, allowing users to review candidate mappings, define manual mappings, and inspect integration-oriented analysis outputs.
