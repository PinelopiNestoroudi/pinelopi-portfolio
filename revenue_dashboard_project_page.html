Project Overview

This project was built as a practical internal reporting platform for revenue performance analysis. It combines role-based access control, configurable filtering, KPI calculations, and interactive charting in a single application flow rather than distributing many separate static files. The project documentation describes it as a Dash-based business dashboard using one main file, one dataset, and custom styling.

What the dashboard does

From dataset and entitlements to guided analysis

  • Reads data from an Excel dataset and applies access control using an entitlements sheet.
  • Uses Flask-based login and user-aware filtering so the visible data depends on the signed-in user.
  • Provides filter groups for industry, business, growth portfolio, client class, and period.
  • Supports KPI cards, period-based analysis, top-client views, donut charts, sector breakdowns, and monthly trend visuals through Plotly.
  • Uses Dash state management to keep filter and UI state consistent across the dashboard experience.

Why it matters

A governed alternative to static distribution

The key value of the project is not only the visual dashboard layer. It is the combination of access-aware data delivery, reusable filtering, and analytical views that make the reporting workflow more maintainable and more useful for ongoing decision support.

Project structure

The dashboard documentation describes a single main application file, Excel-based data and entitlements, custom CSS, responsive layout behavior, and KPI/chart logic implemented directly in the app layer.

Key Capabilities

The project combines business-facing usability with application logic for entitlement filtering, interactive exploration, and recurring performance monitoring.

Access model

Entitlement-based visibility

The code defines a Flask login flow and applies row-level style filtering based on user email and permitted business or industry values, while admin users can see all data.

Analytical interaction

Filter-driven KPI and chart updates

The documented patterns include toggleable filter buttons, period slider behavior, KPI calculations, Plotly charting, and state persistence for a guided analysis experience.

Operational value

Designed for recurring use

The dashboard was built around repeated revenue review workflows, making it easier to inspect performance by client, industry, sector, and month without maintaining multiple separate reporting files.
